Privacy Policy

For gmail-mcp-server — an OAuth client that connects AI agents to the operator's own Gmail accounts.

1. Who this policy applies to

This policy covers gmail-mcp-server, a Model Context Protocol (MCP) server operated by Inder Singh. The server is a single-operator tool: the only Gmail accounts it connects to are those authorized directly by the operator via Google OAuth. It is not a product offered to third parties.

2. What data we access

When you (the operator) grant gmail-mcp-server access to a Gmail account, Google issues an OAuth refresh token scoped to https://www.googleapis.com/auth/gmail.modify. Using that token, the server may:

  • Read message metadata, subjects, bodies, and attachments in the authorized account.
  • List and modify labels, mark messages as read/unread, and archive messages.
  • Send new messages and create drafts from the authorized account.

The server does not access contacts, calendar, Drive, photos, or any non-Gmail Google service.

3. How we use that data

Access is used exclusively to fulfil tool calls made by AI agents authenticated to the server via Microsoft Entra ID. For example, an agent might call search_emails, read_email, or send_email on behalf of the operator. The server does not analyse, aggregate, profile, or monetise message content.

4. How we store data

  • OAuth credentials. Google client_id, client_secret, and refresh tokens are stored as Azure Container Apps secrets (encrypted at rest, not exposed in plaintext environment variables or logs).
  • Message content. Message data fetched from Gmail is returned to the calling agent for the duration of the tool call and is not persisted by the server. The server keeps no database, cache, or log of message content.
  • Request logs. The server logs tool-call metadata (timestamp, caller identity, tool name) for diagnostic purposes only. Logs do not contain message content.

5. How we share data

We do not share Gmail data with any third party. Data leaves the server only:

  • Back to Google (via the Gmail API) when fulfilling a request.
  • To the authenticated AI agent that made the tool call, over a TLS-secured MCP session.

6. How to revoke access

You can revoke the server's access to any Gmail account at any time:

  • Go to myaccount.google.com/permissions.
  • Find gmail-mcp-server (or the OAuth client name) under "Third-party apps with account access."
  • Click Remove Access.

Revocation takes effect immediately. Any subsequent tool call from the server will fail with an invalid_grant error until the account is re-authorized.

7. Security

The server is deployed as a private Azure Container App protected by Entra ID OAuth 2.1 (client-credentials flow with an access_as_agent application role). Each calling agent has its own Entra identity and credentials; there is no shared long-lived password. Credentials and secrets are stored in Azure's managed secret store.

8. Children

gmail-mcp-server is not directed at children and is not used by anyone other than the operator.

9. Changes to this policy

If this policy changes, the "Last updated" date below will change. Because this server has a single operator, there is no notification mechanism beyond the date.
